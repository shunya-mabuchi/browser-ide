# BrowserIDE マルチペイン IDE リデザイン — 実装ロードマップ

> **対応する spec**: [2026-04-27-multi-pane-ide-redesign-design.md](../specs/2026-04-27-multi-pane-ide-redesign-design.md)
> **作成日**: 2026-04-27
> **ステータス**: Week 1 プラン作成済み

## 全体方針

spec を 5 つの Week 単位プランに分割。各 Week の終わりに動作する成果物を出す。Week 1 完了後に Week 2 の詳細プランを書く（イテレーティブ方式）。

## Week 一覧

### Week 1: LLM ライフサイクル再設計 + 4 ペイン基盤

**スコープ**: F（LLM ライフサイクル）+ B（UI シェル）

**達成基準**:
- [ ] エディタが起動 200ms で表示される（ゲート画面廃止）
- [ ] WebGPU 検出 + WASM 自動フォールバック動作
- [ ] ChatPanel が 4 状態（idle / loading / ready / error）で動作
- [ ] モデル切替モーダル動作
- [ ] 4 ペインレイアウト + 縦 Splitter（ファイルツリーは placeholder のまま）
- [ ] Console パネル skeleton（ログ受信は Week 3）
- [ ] localStorage の前回モデル自動復元

**詳細プラン**: [2026-04-27-week1-llm-lifecycle-and-layout.md](./2026-04-27-week1-llm-lifecycle-and-layout.md)

**spec マッピング**:
- セクション 1（レイアウト軸の再設計）
- セクション 6（起動シーケンス + LLM ライフサイクル全体）

### Week 2: OPFS ファイルシステム + ファイルツリー

**スコープ**: A（FsAdapter / FsAdapterSync / WorkspaceStore）+ C（FileTree UI / 右クリック / DnD）

**達成基準（予定）**:
- [ ] OPFS 上に `/workspace/` 階層でファイル CRUD
- [ ] FileTree コンポーネントでフォルダ階層表示
- [ ] 右クリックメニューで新規・リネーム・削除・複製
- [ ] HTML5 DnD でファイル移動
- [ ] 初回 seed（サンプルファイル）動作
- [ ] 800ms 自動保存
- [ ] FsAdapterSync は Worker 側で利用可能な状態にしておく（Week 3 で活用）

**spec マッピング**:
- セクション 2（OPFS とファイルシステム）
- セクション 3 の右クリックメニュー、DnD 部分

### Week 3: タブ管理 + Bundler Worker + Console

**スコープ**: D（TabManager / マルチタブエディタ）+ E（Bundler Worker）+ Console 配信

**達成基準（予定）**:
- [ ] マルチタブで複数ファイル同時編集
- [ ] タブ並び替え（DnD）、close/dirty 表示
- [ ] セッション復元（リロードで完全復帰）
- [ ] Bundler Worker で複数ファイル + esm.sh + import map 解決
- [ ] Preview タブが固定タブとして動作
- [ ] Console パネルに iframe からのログ転送
- [ ] ⌘P クイックオープン

**spec マッピング**:
- セクション 3（エディタ・タブ管理）
- セクション 4（プレビューと Bundling）

### Week 4: AI Diff の前半（パーサー + ChangeSet + 自動オープン）

**スコープ**: G の前半（DiffApplier / jsdiff / 自動オープンフロー）+ ChatPanel リフォーム

**達成基準（予定）**:
- [ ] LLM 応答の `// file:` パーサー + フォールバック
- [ ] ChangeSet 構造とライフサイクル
- [ ] jsdiff で hunk 計算
- [ ] 該当ファイルのタブ自動オープン
- [ ] 自動オープン挙動（応答完了時）
- [ ] 「次回手動」リンクと localStorage 切替

**spec マッピング**:
- セクション 5 の応答フォーマット、パーサー、ChangeSet、自動オープン

### Week 5: CodeMirror inline gutter + Apply フロー + コンテキスト注入

**スコープ**: G の後半（inline gutter UI / Apply）+ H（ContextBuilder）

**達成基準（予定）**:
- [ ] CodeMirror 6 inline gutter で diff 表示
- [ ] 個別 hunk Accept/Reject + ファイル単位 + 全体一括
- [ ] Apply 後の OPFS 書き込み + 再 bundle
- [ ] コンテキスト注入: 暗黙 + `@filename` テキストパース + スマート切り詰め
- [ ] Phase 1 完成基準を満たす

**spec マッピング**:
- セクション 5 の inline gutter、Apply、コンテキスト注入

## イテレーション方針

Week 完了時の振り返りで以下を確認:

1. **想定通りに動いたか**: 達成基準のチェック
2. **次 Week のスコープに影響する学びはあったか**: あれば次 Week プランに反映
3. **spec に齟齬は出たか**: あれば spec 自体を更新（コミット）
4. **Phase 1 の到達感**: 「日常使いできるか」感覚を都度測る
